The Problem

Most AI safety scores can be gamed.
This one cannot.

Typical AI evaluation adds scores together: helpfulness + safety + fluency = total. This lets high performance in one area compensate for violations in another. A system can score 90% while eroding your autonomy — because the math allows trade-offs between things that should never be traded.

Additive (typical)

H + S + F = Score

trade-offs allowed

vs

Multiplicative (kalam)

D = A × L × M

any zero kills the score

How It Works

Four instruments. One measurement.

Gate Function

D = A × L × M. Agency, Legibility, Moral Standing — multiplied, not summed. One zero collapses everything. Non-compensatory by design.

Drift Detection

Continuous monitoring catches when system behaviour quietly diverges from stated values. EWMA/CUSUM early warning before drift compounds.

Built-in Pacing

Configurable cooldown periods for high-stakes decisions. Second-order damping prevents reactive harm. Patience as an engineering constraint.

Agency Score

Quantifies how much decision-making power a system preserves for the human. Measured per-interaction, tracked over time, reported transparently.

Get Started

Clone. Run. Measure.

# Clone the framework

git clone https://github.com/Sternmannli/kalam-framework.git

# Enter and explore

cd kalam-framework

# Run the gate function on a sample interaction

python -m kalam.gate --input sample.json

Python 3.10+. No external dependencies. Works offline.
