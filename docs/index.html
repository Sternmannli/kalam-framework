The Problem

AI safety frameworks add ethics as a sum.
Dignity requires a product.

Additive scoring lets high helpfulness compensate for low consent. A system can score 90/100 while violating autonomy — because the math allows trade-offs between dimensions that should never be traded.

Additive (typical)

H + S + F = Score

trade-offs allowed

vs

Multiplicative (kalam)

D = A × L × M

any zero kills the score

How It Works

Four instruments. One measurement.

Gate Function

D = A × L × M. Agency, Legibility, Moral Standing — multiplied, not summed. One zero collapses everything. Non-compensatory by design.

Drift Detection

Continuous monitoring of semantic drift between stated values and system behavior. KL divergence tracking catches value decay before it compounds.

Thermal Delay

Built-in cooling periods (3–90 days) for high-stakes decisions. Second-order damping prevents reactive harm. Patience as an engineering constraint.

Agency Score

Quantifies how much decision-making power a system preserves for the human. Measured per-interaction, tracked over time, reported transparently.

Get Started

Clone. Run. Measure.

# Clone the framework

git clone https://github.com/Sternmannli/kalam-framework.git

# Enter and explore

cd kalam-framework

# Run the dignity gate on a sample interaction

python -m kalam.gate --input sample.json

Python 3.10+. No external dependencies. Works offline.
